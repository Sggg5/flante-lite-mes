# 脱敏测试数据规则

本仓库暂时不得提交任何真实 Excel 文件。阶段 2 及后续如需增加测试样本，必须先脱敏并确认样本只用于自动化测试或开发演示。

## 严禁提交的数据

- 客户名称、供应商名称、联系人、电话、地址等可识别外部主体的信息。
- 价格、金额、报价、成本、毛利、结算条件等财务或经营敏感数据。
- 真实订单号、合同号、发票号、采购单号、销售单号。
- 真实批次号、炉号、追溯码、条码、仓库库位中可回溯到真实业务的数据。
- 员工姓名、工号、手机号、身份证号、排班记录等员工信息。
- 未脱敏的产品销量、库存、客户需求、生产计划、交付计划等经营数据。

## 允许的测试数据

- 虚拟客户，例如 `CUST-DEMO-001`。
- 虚拟产品编码，例如 `SKU-DEMO-304-001`。
- 虚拟订单号，例如 `SO-DEMO-202607-001`。
- 虚拟批次号，例如 `LOT-DEMO-001`。
- 明确为测试而构造的数量、日期和状态。

## 脱敏要求

- 不能只删除客户名称而保留真实订单号、批次号或金额。
- 数量可保留字段形态，但必须替换为虚拟数量，避免反推出真实业务规模。
- 文件名也要脱敏，不能包含客户、项目、员工或真实订单信息。
- 新增样本前需在 PR 描述中说明样本来源为虚构或已脱敏。

当前阶段仍只做基础框架，不提交 Excel 样本。
